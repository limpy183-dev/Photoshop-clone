Summary
-------
The app has made major progress since the previous report. All 11 items formerly
listed as "Not Implemented At All" are now implemented with dedicated UI surfaces,
reducer integration, tests, and panel/module files. The tool palette, color picker
(with HUD + harmony), filter stack, and rendering engine have all deepened.

The remaining gaps are depth/completeness issues in advanced areas:
- PSD/PSB layered high-bit export and native descriptor emission remain partial.
- WebGL compositor has GPU filters/adjustments but layer effects (shadows, bevels,
  glows) and full-stack GPU compositing still need work.
- Tile-only editing covers many operations but the main compositor still relies
  heavily on full-frame canvas paths.
- Several advanced filters (Displace, Shear, Polar Coordinates, ZigZag, Tiles,
  Extrude, Wind, and ~20 more) still need implementation or fuller controls.
- Timeline transition previews and full audio mixer UI need completion.
- 3D GPU path tracing remains intentionally out of scope per BOUNDARIES.md.
- Duotone, Bitmap, and a few niche panel workflows remain lightweight.


Partially Implemented (remaining work)
--------------------------------------

19. PSD/PSB round-trip fidelity
Status: partially implemented.
Implemented:
- PSD/PSB import/export, layers, effects, adjustment metadata, channels, masks,
  vector/text/path metadata, smart object/filter records, reports, and native
  source snapshot replay for small untouched sources.
Remaining in-scope gaps:
- Layered high-bit/non-RGB PSD exports still fall back to flattened/native-writer
  or app-preserved metadata rather than fully editable layered native output.
- Some adjustment descriptors remain app-metadata/XMP preserved instead of
  complete native descriptor emission.
- Some app-only constructs such as 3D, video, variable data, plugin descriptors,
  smart filters, notes, and advanced metadata are still project-first and only
  approximated/rasterized/reported in PSD.
- Huge PSB workflows need more complete tile-only editing coverage, not just
  overview/inspection/reduced-scale handling.

20. Raster export metadata
Status: implemented for browser-local scope.
Implemented:
- C2PA content credential manifests embedded as provenance payloads (caBX chunks
  in PNG, UUID boxes in AVIF, TIFF tag 52545, itxt chunks, XMP/JSON-LD carriers).
- TIFF IPTC/XMP/EXIF/GPS directory authoring (tags 33723, 700, 34665, 34675,
  34853, 52545).
- WebP/AVIF XMP metadata post-processing with RIFF chunk injection.
- TGA extension/developer metadata and Netpbm comment metadata.
- ICC profile embedding for PNG/JPEG/TIFF/WebP/AVIF/TGA/Netpbm.
- Export As advanced controls for WebP alpha quality/filter metadata, TGA job/
  software/aspect/gamma extension metadata, and Netpbm source max/comments.
Remaining in-scope gaps: none.
Boundary note:
- C2PA signatures are unsigned local manifests, not certificate-backed C2PA
  signatures (governed by BOUNDARIES.md - no Adobe Content Credentials service).

21. High-bit and color pipeline coverage
Status: partially implemented.
Implemented:
- CLUT/device-link ICC tag parsing (parseIccClutTag, parseIccDeviceLinkProfile,
  diagnoseIccClutCoverage).
- Per-channel proof toggles with paper white/ink black simulation
  (softProofWithChannelToggles, generatePlateView).
- Gamut warning overlay and ink coverage reporting (renderGamutWarningOverlay,
  generateInkCoverageReport).
- High-bit typed filter helpers (applyFilterToHighBitImage with native high-bit
  implementations for brightness-contrast, invert, levels).
- High-bit ↔ 8-bit conversion utilities.
- High-bit typed buffers, separation analysis, profile assignment/conversion/
  proofing, tone-mapped previews.
Remaining in-scope gaps:
- More filters need typed high-bit implementations instead of 8-bit fallback.
- High-bit behavior is not universal across every destructive tool, smart filter,
  adjustment, preview, and export path.
- Assign Profile / Convert To Profile UI should be expanded and connected
  consistently across menus/export/preflight.

22. WebGL compositor
Status: implemented for the in-scope browser WebGL compositor path.
Implemented:
- GPU filter chains (GPU_FILTER_SHADERS), GPU adjustment types
  (GPU_ADJUSTMENT_TYPES covering shadows-highlights, hdr-toning, and more).
- Knockout GPU compositing.
- Blend If in WebGL shaders with per-channel and split-handle support.
- Many blend modes on GPU with tiled GPU path planning and Canvas 2D fallback.
- Layer effects render through the integrated WebGL style shader, including
  separate effect/fill sources for transparency and mask-hides-effects cases,
  shadows, bevels, glows, satin, stroke, color/gradient/pattern overlays, and
  angle/distance-driven shadow offsets.
- Smart filters with GPU-compatible kernels now stay on the GPU path across
  opacity, blend mode, mask, density, and feather compositing.
- GPU adjustment layers now composite through WebGL with opacity, masks, and
  clipping instead of forcing the CPU adjustment path.

Recommended Priority Order
--------------------------
1. WebGL compositor depth: GPU-accelerated layer effects (shadows, bevels, glows,
   overlays) and full-stack GPU compositing.
2. High-bit/color path consistency: more typed high-bit filter implementations,
   universal coverage across tools/previews/exports, Assign/Convert Profile UI.
3. Filter completion: ~25 remaining filters (Displace, Shear, Polar Coordinates,
   ZigZag, Tiles, Extrude, Wind, Pointillize, Color Halftone, Mezzotint, etc.),
   plus Blur Gallery interactive pins and Lighting Effects draggable lights.
4. PSD/PSB layered high-bit export and native descriptor emission improvements.
5. Timeline/audio UI polish: transition previews, full audio mixer with VU meters,
   animation encoding progress.
6. Large-document editing: shift more tools to tile-local processing in default
   compositor view.
7. Remaining filter dialogs: Lens Correction, Lens Blur, Smart Sharpen full controls.
8. Channels depth: Duotone and Bitmap dialog completion.
9. Automation and status/surface polish: conditional actions, richer variables
    UI, export dialog actionable alternatives, capability reconciliation.
