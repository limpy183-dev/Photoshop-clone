CONCRETE REMAINING GAPS
--------------------------------------------------------------------------------

[MEDIUM] 1. Photomerge — Boundary Warp slider missing
    File:  components/photoshop/photomerge-dialog.tsx
    Detail: Cylindrical + spherical projection and lens distortion correction
            are wired up (geometricCorrection, lensModel, vignetteRemoval).
            However, the "Boundary Warp" parameter — which lets users pull
            transparent edges inward to fill a rectangular crop — is not
            exposed as a UI slider. The seam-search budget constant
            (`boundaryCandidateBudget`) is internal-only.
    Effort: ~80 lines (add slider 0–100, pipe into composition stage).

[MEDIUM] 2. Image Processor — no watermark / copyright overlay
    File:  components/photoshop/processing-dialogs.tsx (ImageProcessorDialog)
    Detail: Resize + format-convert + matte are wired. Missing: text/image
            watermark overlay (e.g., © string in corner), and IPTC/XMP
            copyright metadata embedding on output. `grep watermark|copyright`
            returns no hits in this file.
    Effort: ~120 lines (watermark text input + position selector + canvas
            draw + IPTC metadata write).

[MEDIUM] 3. Print dialog exposes only 2 of 6 mark types
    Files: components/photoshop/print-engine.ts — defines six PrintMarkType
           values: registration, corner-crop, center-crop, description,
           labels, color-bars (all rendered correctly in renderPrintMarks).
           components/photoshop/advanced-subsystems-dialog.tsx
           (PrintWorkspace) — UI exposes only `cropMarks` and
           `registrationMarks` checkboxes.
    Detail: Center-crop marks, color bars, page description text, and labels
            are implemented in the engine but unreachable from the UI.
    Effort: ~60 lines (add four checkboxes + propagate to settings).

[MEDIUM] 4. Transform Selection — no on-canvas drag handles
    Files: components/photoshop/canvas-view.tsx (transformHandles ~line 8543)
           components/photoshop/editor-context.tsx (line ~3553)
           components/photoshop/management-dialogs.tsx (line ~346)
    Detail: Free-transform on layers has full 8-handle + rotate-handle drag
            support. Select ▸ Transform Selection is currently parameter-driven
            (scale + rotation entered via a dialog). It needs the same eight
            corner/edge handles drawn on the marquee outline that scale/rotate
            the boundary independently of the underlying pixels.
    Effort: ~250 lines (handle hit testing + drag math + matrix application
            to selection mask only).

INTENTIONAL NON-GOALS (per BOUNDARIES.md, not gaps)
--------------------------------------------------------------------------------

The following will not be implemented and should not appear in any gap report:
  - Adobe Firefly / Sensei AI services
  - Native plugin runtimes (.8bf format)
  - CMM-certified ICC color management
  - MP4 / H.264 video export (only MediaRecorder formats)
  - 16/32-bit display preview (browser canvas is 8-bit RGBA only)
  - Server-side state / cloud sync / Creative Cloud
  - Exact native-Photoshop algorithm parity at pixel level
