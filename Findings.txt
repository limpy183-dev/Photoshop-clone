Prioritized Findings
P0, verified defect: reducer executes twice despite being impure. dispatch manually calls reducer, then rawDispatch runs it again (editor-context.tsx (line 4451)). The reducer generates IDs/timestamps (editor-context.tsx (line 2145)) and schedules mutable asynchronous compression (editor-context.tsx (line 2719), editor-context.tsx (line 470)). This can diverge stateRef from React state and schedule overlapping compression against shared snapshots, risking corrupted dimensions and leaked blobs.
Improve: make the reducer deterministic and side-effect-free; create IDs/timestamps before dispatch and move compression/release into a history service.

P0, verified defect: saving can mark unsaved concurrent edits as saved. The document is serialized before asynchronous file operations (menu-bar.tsx (line 1913)), but completion later calls markDocumentSaved (menu-bar.tsx (line 1965)), which records the history index at completion time (editor-context.tsx (line 3715)). Edits made while the picker/write is pending can therefore appear saved although they are absent from disk.
Improve: capture the document revision at serialization time and pass that exact revision in the save-completed action; remain dirty when the current revision differs.

P1, verified defect: delayed history restore is not document-bound. Undo captures a document and entry, asynchronously decompresses it, then dispatches an action without a document ID (editor-context.tsx (line 4758), editor-context.tsx (line 4792)). The reducer applies it to whichever document is active at completion (editor-context.tsx (line 2807)). Switching documents during decompression can restore old layers into another document.
Improve: include docId and expected history generation in restore actions and discard stale completions.

P1, verified defect: history does not cover persisted document state. PsDocument contains timeline, plugins, metadata, color management, and print settings (types.ts (line 1823)), but HistoryEntry and makeHistoryEntry omit most of them (types.ts (line 1888), editor-context.tsx (line 3926)). File Info modifies three omitted fields and explicitly creates a history entry (file-info-dialog.tsx (line 161)); undo cannot revert them.
Improve: define one complete document snapshot contract or use inverse domain operations instead of a manually duplicated subset.

P1, verified defect: dirty tracking has false negatives and false positives. A manually maintained action allow-list controls dirty state (editor-context.tsx (line 1676)). Persisted set-plugin-storage mutations are omitted and occur without a history commit (advanced-subsystems-dialog.tsx (line 2562)). Conversely, listed actions mark dirty even when the reducer rejects them, such as removing the only layer (editor-context.tsx (line 2128)).
Improve: derive dirty state from actual per-document revision changes, not action names.

P1, verified defect: save events can target the wrong document. Shortcut saves omit docId (use-shortcuts.ts (line 157)); completion re-emits that missing ID (menu-bar.tsx (line 1977)). The asset generator then falls back to the currently active document (image-assets-generator-runner.tsx (line 210)). Switching tabs during save can run generation for the wrong document.
Improve: every request/completion should carry resolved docId plus a correlation ID.

P2, verified defect: the editor Home workspace is never rendered. It is lazy-loaded (editor-app.tsx (line 57)), but the state value is discarded (editor-app.tsx (line 111)); ps-show-home only updates that unused state (editor-app.tsx (line 231)). With no documents, CanvasView displays plain text instead (canvas-view.tsx (line 7013)).
Improve: render HomeWorkspace when homeOpen || !activeDoc.

P2, architecture risk: global context and module boundaries amplify coupling. One context exposes nearly every state slice and command (editor-context.tsx (line 4089)); its value depends on nearly all state (editor-context.tsx (line 5395)). Any change invalidates all useEditor consumers. The same module also imports UI, storage, algorithms, playback, history, and rendering services (editor-context.tsx (line 3)).
Improve: split document/history/tool/lifecycle stores with selector subscriptions; separate command handlers from reducers and move the close dialog into the shell.

3rd:

Findings
[P0] Tile eviction can silently destroy pixel data. tile-store.ts (line 402) clears resident data even when OPFS is disabled or persistence fails. tile-store.ts (line 262) later reconstructs an empty tile. Only evict after confirmed persistence; otherwise retain the tile or reject the allocation.

[P1] OPFS tile keys collide across documents. tiled-backing-store.ts (line 206) and tiled-backing-store.ts (line 235) omit a document/store namespace. PSB tiles also use fixed identifiers at psb-tile-view.ts (line 5). Include a unique document namespace and scratch-format version in every key.

[P1] Pointer-to-document conversion is incorrect in tile-only and rotated views. Tile mode makes the composite canvas viewport-local at canvas-view.tsx (line 727), but canvas-view.tsx (line 1052) maps its bounds directly to full-document dimensions. CSS rotation at canvas-view.tsx (line 7093) is also not inverted. Centralize screen/document transforms using an inverse DOMMatrix plus viewport origin.

[P1] Large-document fallback can still allocate full-document canvases. The overlay always uses full document dimensions at canvas-view.tsx (line 7108). Incompatible documents disable tiling at tile-only-pipeline.ts (line 792), followed by a full-size allocation at canvas-view.tsx (line 752). Use viewport-sized overlays and never silently fall back to an unsafe full canvas.

[P1] Tile rendering applies smart filters twice and lacks effect halos. layer-tile-renderer.ts (line 402) materializes smart filters, then tile-only-pipeline.ts (line 530) applies them again. Filters operate on cropped tiles without neighboring pixels at tile-only-pipeline.ts (line 462), causing seams and stale adjacent tiles. Define one render contract and propagate effect radius through dirty-tile invalidation.

[P1] Filter cancellation converts canceled work into main-thread work. Abort rejection at filter-worker.ts (line 1534) reaches a catch that permanently marks the worker failed and runs the filter synchronously at filter-worker.ts (line 1546). Preserve AbortError, implement worker cancellation, clean listeners, and serialize worker initialization.

[P1] WebGL resources and contexts churn without deterministic cleanup. Composition creates transient canvases/contexts at canvas-view.tsx (line 862) and webgl-compositor.ts (line 3004). Shader passes allocate programs, buffers, and textures around webgl-compositor.ts (line 1851), but cleanup at webgl-compositor.ts (line 1978) covers only extra textures. Reuse a persistent compositor and add try/finally disposal and context-loss handling.

[P1] WebGL capability planning assumes a 16,384-pixel texture limit. webgl-compositor.ts (line 1145) uses a guessed limit rather than MAX_TEXTURE_SIZE; allocations lack robust framebuffer/error checks. Query the active context and downgrade cleanly when texture or framebuffer allocation fails.

[P1] Brush workflows allocate multiple full-document canvases per stroke. Transparency locking and buffered strokes clone full layers at canvas-view.tsx (line 1273) and canvas-view.tsx (line 1310). An 8192² RGBA canvas is roughly 256 MiB. Use stroke-bounds or tile-local scratch surfaces and account for them in the memory budget.

[P1] “Tiled” filters remain full-document and block rendering. filter-worker.ts (line 1663) allocates a full output ImageData; row processing repeatedly uses slice() and only yields through resolved promises at filter-worker.ts (line 1668), which does not yield to input/rendering. Stream bounded tiles and use task/macrotask scheduling.

[P2] Progressive rendering is not meaningfully integrated and has a restart race. Canvas handling at canvas-view.tsx (line 832) ignores planned preview scale and tile keys. An older run can clear a newer runner’s state in progressive-preview.ts (line 178). Guard cleanup with a generation/controller identity and wire the planner to actual low-resolution rendering.

[P2] Pointer cancellation and lost capture are unhandled. The canvas registers down/move/up but no pointercancel or lostpointercapture handlers at canvas-view.tsx (line 7075). Interrupted input can retain stroke state and scratch canvases. Add one idempotent cancellation path for cancel, lost capture, blur, and unmount.

[P2] DPR changes do not repaint the custom cursor. canvas-view.tsx (line 7003) reads devicePixelRatio without observing later monitor or zoom changes. Track DPR through resolution-media/resize events and resize the backing canvas when it changes.

[P2] The canvas pool retains oversized backing stores indefinitely. canvas-utils.ts (line 111) returns large canvases to the pool, while cleanup at canvas-utils.ts (line 125) has no production caller. Cap by retained bytes, zero evicted dimensions, and clean on document close or memory pressure.

The broader maintainability problem is duplicated rendering semantics across the 2D, WebGL, tile-only, and layer-tile paths. A canonical layer/render graph with backend-specific execution would prevent divergences such as double filters and inconsistent fallback behavior.

High — Public generative-fill relay can expose a billable/private upstream
The route has no authentication, rate limit, origin validation, request schema, body-size limit, timeout, or response-size limit. It forwards attacker-controlled JSON using the server’s bearer secret.
Exploitability: requires a server deployment with GENERATIVE_FILL_ENDPOINT and GENERATIVE_FILL_API_KEY configured. An unauthenticated remote attacker could consume upstream capacity or cost.
References: app/api/photoshop/generative-fill/route.ts (line 3), route.ts (line 13), route.ts (line 14), route.ts (line 22)
Medium — Marketing rate limits are trivially bypassable
Without trusted-proxy mode, the client identity is derived entirely from caller-controlled browser headers. Rotating User-Agent bypassed the limiter in dynamic testing. Trusted mode can also accept spoofed forwarding headers unless the proxy strips all incoming copies. Buckets are process-local, so restarts and multiple instances weaken enforcement.
Exploitability: unauthenticated direct HTTP access permits quota exhaustion, data poisoning, and memory growth through unique bucket keys.
References: lib/marketing-store.ts (line 96), marketing-store.ts (line 174), feedback/route.ts (line 42), subscribe/route.ts (line 46)
Medium — Advanced image decoders validate dimensions after allocation
TIFF, EXR, HEIC, JPEG 2000, and RAW inputs are decoded before canvas-dimension limits are checked. The compressed 80 MB file limit does not constrain decompressed memory.
Exploitability: a user must import a crafted file accepted by the relevant codec. Impact is browser-tab memory exhaustion or prolonged CPU usage.
References: raster-codecs.ts (line 544), raster-codecs.ts (line 579), raster-codecs.ts (line 631), raster-codecs.ts (line 722), raster-codecs.ts (line 773), advanced-subsystems.ts (line 30)
Medium — Project imports permit parallel decode amplification
Image data URLs are decoded before dimensions are checked. Layers and channels are decoded concurrently, while each layer may contain an uncapped number of smart-filter masks.

References: document-io.ts (line 3092), document-io.ts (line 3120), document-io.ts (line 3191), menu-bar.tsx (line 2081)

Recent documents store complete serialized projects, including canvas and mask data URLs, in localStorage. Persistence hardcodes eight documents and does not consume the preference that allows users to select zero.
Exploitability: requires same-browser-profile access, a malicious extension, local account access, or same-origin script execution. Sensitive image contents remain after closing the document.

[P1] Imported 16/32-bit PSDs lose precision on save. Import renders layers into 8-bit browser canvases, while saving merely expands those 8-bit values back to the declared depth. [P1] Every high-bit or non-RGB PSD is routed through a structure-reducing writer. Groups are dissolved and masks, adjustment descriptors, guides, channels, paths, notes, print settings, linked files and most image resources are omitted. Index-based metadata restoration can then attach metadata to the wrong layer. document-io.ts (line 4155) psd-native-writer.ts (line 563) psd-native-writer.ts (line 610) psd-compatibility.ts (line 425)

[P1] Original ICC data is discarded and replaced by a synthetic, identification-only profile. The importer extracts profile bytes but retains only a mapped name. [P1] HDR highlights are clipped before float TIFF/EXR export.
